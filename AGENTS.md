# AGENTS.md

Этот файл описывает правила взаимодействия пользователя с Codex при работе над проектом. Он не относится к runtime-логике приложения, но должен учитываться при планировании, редактировании, коммитах, push и дальнейшей разработке.

## Обязательные правила

1. Не выполнять `git push` без явного согласия пользователя.
2. `git commit` можно выполнять самостоятельно, если изменения логически завершены.
3. Сообщения коммитов писать на русском языке.
4. После каждого `git push`, выполненного с явного согласия пользователя, перечитывать этот файл как дополнительный контроль правил.
5. Непосредственно перед началом каждой отдельной задачи, затрагивающей исходный код, полностью перечитывать этот файл. Серия связанных изменений в рамках одного непрерывного шага считается одной задачей и не требует повторного чтения перед каждым файлом.

## Правила написания кода

- Новый и изменяемый публичный Python-код сопровождать docstrings по аналогии с JavaDoc/KotlinDoc.
- Для docstrings использовать Google style и соблюдать общие рекомендации PEP 257.
- Docstring должен объяснять назначение класса, функции или метода. Для нетривиального интерфейса также описывать аргументы, возвращаемое значение и ожидаемые исключения.
- Значения enum, статусы и другие неочевидные константы сопровождать краткими комментариями, объясняющими их смысл.
- Сложные участки реализации комментировать там, где важно объяснить причину решения, ограничение или неочевидное поведение.
- Не добавлять комментарии, которые только дословно пересказывают очевидную операцию кода.
- Комментарии и docstrings в коде этого проекта писать на русском языке, сохраняя английские имена идентификаторов и общепринятые технические термины.

## Рабочий порядок

- Перед действиями с Git проверять статус рабочей копии.
- Коммиты можно делать без отдельного подтверждения, но push всегда требует явного согласия пользователя.
- Если правило из этого файла конфликтует с текущим планом, сначала уточнить действие у пользователя.
